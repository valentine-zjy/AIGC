# Story 4.1: 一键优化入口与护栏审查
Status: done

## Story

As a 用户,
I want 在启动一键优化前先看到范围、适用条件、风险与可回退说明,
so that 我可以在知情前提下决定是否进入批量优化预览。

## Acceptance Criteria

1. **Given** 当前任务满足一键优化条件
   **When** 用户进入一键优化入口
   **Then** 系统展示护栏审查面板
   **And** 清楚说明本次预览范围、适用对象、前置条件和主要后果
2. **Given** 当前任务不满足一键优化条件
   **When** 用户尝试触发一键优化
   **Then** 系统阻止启动
   **And** 明确给出不可启动原因与下一步建议
3. **Given** 用户查看护栏审查面板
   **When** 面板展示关键说明
   **Then** 系统说明本次操作的可回退性、与后续精修的关系以及潜在影响
4. **Given** 用户确认启动一键优化
   **When** 系统接收确认动作
   **Then** 系统创建对应的一键预览会话
   **And** 通过异步状态推进预览，而不是阻塞当前页面
5. **Given** 审查面板存在多个动作入口
   **When** 用户查看按钮层级
   **Then** 同一区域只保留一个最强主操作
   **And** 返回、取消与危险动作不与主操作抢层级
