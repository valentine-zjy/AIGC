# Story 4.2: 一键优化结果预览与变化摘要
Status: done

## Story

As a 用户,
I want 在最终导出前预览一键优化结果并查看变化摘要,
so that 我可以判断批量预览是否符合预期，再决定是否继续精修。

## Acceptance Criteria

1. **Given** 一键优化预览已完成
   **When** 用户进入结果预览区域
   **Then** 系统展示优化后的预览结果
   **And** 用户可以在导出前查看本次批量预览带来的主要变化
2. **Given** 用户查看结果摘要
   **When** 页面展示变化摘要卡
   **Then** 系统汇总本次处理的段落数、剩余项与关键改动
   **And** 这些信息足以支持用户判断是否接受当前结果
3. **Given** 一键优化仍在处理中、失败或暂时没有可预览内容
   **When** 用户打开预览区域
   **Then** 系统展示清晰的处理中、失败或空态反馈
   **And** 告知当前状态及下一步可执行动作
4. **Given** 用户查看一键优化结果
   **When** 需要理解与后续流程的关系
   **Then** 系统明确提示可以继续进入局部精修
   **And** 不要求用户在预览阶段立即导出或结束任务
5. **Given** 用户通过键盘或辅助技术浏览预览与摘要
   **When** 页面展示结果、摘要和动作入口
   **Then** 信息与控件具备清晰语义和可读反馈
