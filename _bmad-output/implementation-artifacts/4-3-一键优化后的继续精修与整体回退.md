# Story 4.3: 一键优化后的继续精修与整体回退
Status: done

## Story

As a 用户,
I want 在一键优化预览后继续进入段落级精修，并在不满意时整体回退到预览前状态,
so that 批量优化只是加速步骤，而不是不可逆的最终结果。

## Acceptance Criteria

1. **Given** 用户已完成一键优化预览
   **When** 用户选择继续精修
   **Then** 系统将当前任务带回段落级工作台
   **And** 用户无需重新上传文档或重新创建任务
2. **Given** 用户认为一键优化结果不符合预期
   **When** 用户触发整体回退
   **Then** 系统展示回退确认与影响说明
   **And** 明确说明回退范围、可恢复性与对当前状态的影响
3. **Given** 用户确认整体回退
   **When** 系统执行回退
   **Then** 当前任务恢复到一键优化前的可继续编辑状态
   **And** 用户仍可重新启动 one-click 或继续段落级处理
4. **Given** 一键优化与回退动作需要被追踪
   **When** 相关状态落盘
   **Then** 系统以 append-only 事件日志方式保留证据
   **And** 为历史查看、排障和后续扩展保留依据
5. **Given** 用户从预览进入继续精修
   **When** 工作台进入 `completed + rewrite` 状态
   **Then** 段落列表、诊断面板、改写建议、采纳/反馈/局部回退与历史面板都仍可继续使用

## Dev Notes

- 4.3 的关键修复点不是 UI，而是行为链路：原实现把任务推进到 `rewrite` 后，段落级服务仍只接受 `scan`，会导致“继续精修”看得到但用不了。
- 本次通过 `isParagraphWorkbenchReady()` 统一工作台可用判定，确保 one-click preview 完成后仍能进入局部精修。
- 整体回退目前回退的是 preview session 与任务阶段，不会误清除用户后续已经做出的段落级决定。
